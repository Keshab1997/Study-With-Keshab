# Content-to-JSON Automation Plan

## Goal
Create an automated system that converts formatted Bengali mathematics content into structured JSON files, eliminating manual JSON creation.

## Current State Analysis
- Existing JSON structure in `subject/Math/Age/data/`
- Content follows a specific pattern with chapters, rules, steps, examples
- Manual conversion is time-consuming

## Proposed Solution

### 1. Create a Content Parser
- **Input**: Formatted Bengali text with markdown-like structure
- **Output**: Valid JSON matching existing schema
- **Features**:
  - Parse headings (##, ###) as chapter/rule titles
  - Parse bold/italic markers
  - Handle mathematical expressions
  - Auto-generate section types (title, box, etc.)

### 2. Template System
```
# Class 4: Chapter Name (English Translation)
## Rule 1: Rule Name (English)
**Content in Bengali with **bold** markers**
- Bullet points
1. Numbered steps
## Rule 2: ...
```

Would convert to:
```json
{
  "chapterName": "Chapter Name (English Translation)",
  "classNumber": "Class 4",
  "sections": [
    { "type": "title", "content": "Chapter Name..." },
    { "type": "box", "content": "<strong>Rule 1...</strong>" },
    ...
  ]
}
```

### 3. Implementation Steps
1. Create parser script (`/scripts/content-to-json.js`)
2. Define content format specification
3. Add validation to ensure valid JSON output
4. Create documentation for content authors

### 4. Folder Structure
```
scripts/
  content-to-json.js     # Main parser
  template.md           # Content template
  validate-json.js      # JSON validator

docs/
  content-format.md     # Format specification
```

### 5. Usage Workflow
1. Author writes content in `.md` format using template
2. Run `node scripts/content-to-json.js input.md output.json`
3. Review and validate output
4. Place in appropriate data folder
